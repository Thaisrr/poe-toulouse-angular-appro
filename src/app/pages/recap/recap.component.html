Récapitulatif des Framework

Angular Vue React
Type de framework Framework opinionated : il y a la façon de faire Angular. Framework léger, propose plusieurs façon de développer : application front, intégré dans une page HTML, dans wordpress,... Plus une librairie qu'un framework : permet surtout de créer du composants avec du JSX.
Interpolation et Data Binding Automatique, tous les attributs de classe du composant sont surveillées, et le DOM mis à jour à chaque modification. Le DOM est mis à jour pour les données surveillées par React uniquement : les states ( ou les hooks comme useState ) Les données surveillées sont, pour l'Option API, dans la méthode Data, et pour la composition API dans les states reactifs ( reactive() ou ref() )
Language Typescript Uniquement Javascript par défaut, mais Typescript OK sans soucis Javascript par défaut, mais Typescript OK pour Vue 3 ( galère sur Vue2 )
Template Fichier HTML OU attribut template. Accepte du HTML + des directives Angular JSX : Javascript XML, syntaxe entre le XML et le javascript. Balise template ou fichier HTML, accepte du HTML + des directives Vue.
Ecosystème Angular est + lourd, mais tout est présent : RXJS, le Router, l'injection de dépendances, ... Très léger mais ne comprend que le nécéssaire pour créer des composants. Il faut installer des librairies pour à peut prét tous le reste. Très léger, quelques outils par défaut, mais uniquement le stric minimum
Réactivité des données Basé sur l'injection de services, et RXJS ( Observables, Subjects,... ) Le Context, ou le store Redux ( Flux est KO ) Le store Vuex ( pour vue 2 ), ou Pinia ( pour vue 3 )
Réagir à la modification de données Observable, Subject, événements natifs : ngModelChange, ngOnChange, ... UseEffect -> ajout des données à surveiller dans le tableau de dépendances Watchers -> on peut créer 1 watcher par donnée à surveillée
Requêtes Observables avec RXJS, avec HttpClient ( à importer avec HttpClientModule ) Promesses avec Axios ou Fetch ( mais peut intégrer RXJS ) Promesses avec Axios ou Fetch ( mais peut intégrer RXJS )
Directives *ngIf, *ngFor, *ngSwitch, [(ngModel)], ... Aucune : uniquement du Javascript v-if, v-for, v-model, v-switch, v-show,...
